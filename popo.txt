ლაბორატორიული სამუშაო: Bastion Host-ის კონფიგურაცია AWS-ში

- ბაზისური ცოდნა VPC და EC2 სერვისებში
- SSH კლიენტი (PuTTY ან Terminal)

 ნაბიჯი 1: VPC-ის მომზადება
1. შექმენით ახალი VPC (CIDR: 10.0.0.0/16)
2. შექმენით ორი ქვექსელი:
   - საჯარო ქვექსელი (Public Subnet - 10.0.1.0/24)
   - პრივატული ქვექსელი (Private Subnet - 10.0.2.0/24)
3. დააკონფიგურირეთ Internet Gateway და მიაბით VPC-ს
4. შექმენით და დააკონფიგურირეთ Route Tables საჯარო და პრივატული ქვექსელებისთვის

 ნაბიჯი 2: უსაფრთხოების ჯგუფების შექმნა
1. შექმენით Bastion Host-ის უსაფრთხოების ჯგუფი (bastion-sg):
   - დაუშვით SSH (port 22) თქვენი IP მისამართიდან
2. შექმენით პრივატული ინსტანსის უსაფრთხოების ჯგუფი (private-sg):
   - დაუშვით SSH (port 22) მხოლოდ bastion-sg-დან

ნაბიჯი 3: EC2 ინსტანსების გაშვება
1. Bastion Host-ის შექმნა:
   - Amazon Linux 2 AMI
   - t2.micro ტიპი
   - საჯარო ქვექსელი
   - ავტომატური საჯარო IP მინიჭება
   - bastion-sg უსაფრთხოების ჯგუფი
   - შექმენით ახალი key pair: bastion-key

2. პრივატული ინსტანსის შექმნა:
   - Amazon Linux 2 AMI
   - t2.micro ტიპი
   - პრივატული ქვექსელი
   - private-sg უსაფრთხოების ჯგუფი
   - გამოიყენეთ იგივე key pair: bastion-key

ნაბიჯი 4: SSH კონფიგურაცია და ტესტირება
1. დააკონფიგურირეთ SSH config ფაილი ლოკალურ მანქანაზე:

# ~/.ssh/config
Host bastion
    HostName <bastion-public-ip>
    User ec2-user
    IdentityFile ~/.ssh/bastion-key.pem

Host private
    HostName <private-instance-private-ip>
    User ec2-user
    IdentityFile ~/.ssh/bastion-key.pem
    ProxyCommand ssh -W %h:%p bastion


2. დაამოწმეთ კავშირი:
   - შეამოწმეთ Bastion Host-თან კავშირი: `ssh bastion`
   - შეამოწმეთ პრივატულ ინსტანსთან კავშირი: `ssh private`

 ნაბიჯი 5: ტესტირება და ვალიდაცია
1. შეამოწმეთ ინტერნეტთან წვდომა პრივატულ ინსტანსზე:

ping 8.8.8.8
curl ifconfig.me


2. დაადასტურეთ უსაფრთხოების კონფიგურაცია:
   - სცადეთ პირდაპირი წვდომა პრივატულ ინსტანსთან (არ უნდა შესრულდეს)
   - შეამოწმეთ წვდომა Bastion-ის გავლით
